# External Proof Phase

The external proof phase begins when synthetic development stops increasing conviction.

## Acceptable Proof

- target-user feedback,
- real or public input data,
- workflow validation,
- pilot or sample-data request,
- willingness-to-pay signal,
- real artifact judged useful.

## Required Packet

- `EXTERNAL_PROOF_PACKET.md`
- `CUSTOMER_FEEDBACK.md`
- `REAL_DATA_RECEIPT.md` when applicable
- `UPDATED_IC_MEMO.md`
- `FUNDING_RECOMMENDATION.md`

## Final Bar

A repo advances only if it can support this sentence:

> A real target user saw this, understood it, and wanted the output.
