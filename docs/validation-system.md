# Validation System

The validator is responsible for rejecting product-shaped theater.

## Checks

- required product files exist,
- `make demo`, `make test`, `make eval`, and `make receipt` pass,
- app code has enough product logic,
- product surface exists,
- generated buyer artifacts exist,
- mocks and assumptions are explicit,
- investment-readiness questions are answered,
- Taste Bank calibration exists for funded work.

## Important Distinction

Validator readiness is not fundability. It means the repo is mature enough to be judged.
