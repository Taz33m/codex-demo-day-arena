# Operating Model

The arena operates like a startup foundry, not a hackathon.

## Loop

1. Select product candidates.
2. Launch repo-owner agents.
3. Validate product maturity.
4. Score buyer usefulness.
5. Allocate compute to top candidates.
6. Run proof sprints.
7. Stop when synthetic development no longer changes the funding decision.
8. Require external proof before any final investment decision.

## Stopping Rule

Teams stop developing when the company is judgeable, not when the product is finished.

After a repo has a product surface, receipt, evals, mock disclosure, and buyer-useful artifact, more local development is treated as suspect unless it changes an investment belief.

## Portfolio Rule

Only fundable products receive more compute. Once all strong products require external evidence, the arena closes.
