# External Proof Protocol

The arena has reached the synthetic ceiling: validator readiness and local buyer judges are no longer enough to increase conviction.

A repo can advance from buyer-compelling to investment-ready only with external evidence.

## Acceptable Proof

Acceptable proof includes at least one of:

1. real or public input data,
2. target-user feedback,
3. buyer workflow validation,
4. willingness-to-pay signal,
5. pilot or sample-data request,
6. demo video reaction,
7. real artifact judged useful by a target user.

## Not Enough

These do not move a company toward the fictional $10M check:

- synthetic validator readiness,
- local coding without external input,
- generated IC memos,
- prettier demos,
- additional fake fixtures,
- internal evals that do not use real or public inputs,
- another synthetic buyer judge.

## Required Proof Packet

Any repo seeking to move from `external-proof-required` to IC consideration must produce:

- `EXTERNAL_PROOF_PACKET.md`
- `CUSTOMER_FEEDBACK.md`
- `REAL_DATA_RECEIPT.md` when real or public input data is used
- `UPDATED_IC_MEMO.md`
- `FUNDING_RECOMMENDATION.md`

## Advancement Standard

A repo advances only if the packet can support this sentence:

> A real target user saw this, understood it, and wanted the output.

If the repo cannot support that sentence, freeze normal development and keep the product in external-proof-required status.
