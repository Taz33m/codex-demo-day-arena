# Proof Priority Rubric

External proof should not be attempted across every repo at once. Rank candidates by proof feasibility, not just buyer score.

## Formula

External Proof Priority =

`buyer pain + reachable customer access + available real/sample data + demo clarity + willingness-to-pay signal - integration complexity - compliance/data sensitivity`

Each factor is scored from 0 to 10.

## Factors

- Buyer pain: urgency, frequency, and cost of the workflow.
- Reachable customer access: how quickly we can reach a real target user.
- Available real/sample data: whether public, personal, anonymized, or buyer-provided inputs can be used quickly.
- Demo clarity: whether the value is visible without explanation.
- Willingness-to-pay signal: whether the workflow plausibly supports budget, pilot, or paid usage.
- Integration complexity: how much setup is required before proof can happen.
- Compliance/data sensitivity: whether proof requires sensitive, regulated, confidential, or personally identifiable data.

## Categories

- Proof-now finalists: can seek real-world proof immediately.
- Proof-later contenders: promising, but proof path needs more access, better data, or a warmer buyer.
- Customer-data-required: cannot progress honestly without a partner, customer export, sandbox credential, or sensitive workflow sample.
- Synthetic-ceiling-reached: should not receive more work until an outside signal appears.

## Current Rule

Only proof-now finalists get immediate work. Passing the validator is table stakes; external proof is the new funding gate.
