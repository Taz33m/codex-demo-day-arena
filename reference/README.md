# Taste Bank

The Taste Bank gives repo-owner agents concrete examples of what serious product quality feels like.

It is intentionally small. It is not a reference archive, a template library, or a permission slip to copy another product. It is a calibration layer for product taste before an owner spends another round of coding effort.

Core rule:

> Borrow the bar, not the build.

Before a funded owner sprint, read:

1. `reference/TASTE_BANK_RUBRIC.md`
2. At least two relevant exemplar profiles under `reference/exemplars/`
3. At least one relevant anti-pattern under `reference/anti-patterns/`

Then write `TASTE_CALIBRATION.md` in the product repo before modifying code.

The calibration should answer:

1. Which reference exemplars are most relevant?
2. What standards should this repo borrow?
3. What should it explicitly not copy?
4. What is this repo's equivalent of the reference product's aha moment?
5. What would make this product feel toy-like?
6. Which anti-pattern is this repo most at risk of?
7. What product-quality bar must this sprint clear?

Do not copy product ideas, branding, UI layouts, or implementation. Use these references to calibrate ambition, product density, workflow clarity, artifact quality, repo seriousness, and evaluation rigor.
